# iFocal

iFocal 是一款面向 Chrome / Edge 的 Manifest V3 浏览器扩展，核心定位是“网页内翻译 + 独立 AI 助手窗口”。它把划词翻译、悬浮翻译、网页全文翻译和多助手对话放在同一个设置体系里，适合需要在浏览器内频繁阅读、翻译和总结网页内容的场景。

当前实现以开发者模式加载为主，尚未提供商店发布包。

## 当前能力

### 翻译

- **划词翻译**：选中文本后显示翻译圆点，点击后在浮层中展示译文。
- **悬浮翻译**：按住触发键（默认 `Alt`）对当前块级内容翻译，可选择浮层显示或插入到原文后。
- **全文翻译**：通过右键菜单或快捷键触发，使用机器翻译渠道批量翻译当前页面，支持显示原文 / 显示译文切换。
- **AI / 机器翻译模式**：划词和悬浮翻译可配置使用 AI 翻译或机器翻译；全文翻译默认面向机器翻译并发场景。
- **译文样式预设**：支持点状下划线、高亮、模糊、波浪线、弱化、边框、分割线、斜体、加粗以及自定义 CSS。
- **页面隔离**：内容脚本使用 Shadow DOM 承载浮层 UI，并对插入页面的译文节点做样式命名隔离。

### 助手

- **独立助手窗口**：点击扩展图标或按 `Ctrl+Shift+O` 打开，窗口保持单例。
- **内置助手**：默认提供聊天、翻译、总结三个助手，也支持添加自定义助手、图标、模型和提示词。
- **会话历史**：会话保存在 `chrome.storage.local`，支持按助手切换、删除和数量上限控制。
- **流式响应**：通过 `chrome.runtime.connect({ name: 'streaming' })` 与 background 通信，实时显示模型输出。
- **上下文与思考模式**：可按任务配置上下文消息数量、流式开关、思考模式和 reasoning effort。
- **文件附件**：当前主要支持图片作为多模态输入传给 OpenAI / OpenAI-compatible Chat Completions 兼容接口。
- **剪贴板监听**：可配置自动读取剪贴板内容填入助手输入框。

### 设置

- **AI 渠道**：支持 OpenAI、Google Gemini、OpenAI-compatible 三类渠道。
- **模型命名**：模型可写成 `modelId#displayName`，请求时使用 `modelId`，界面显示 `displayName`。
- **机器翻译渠道**：内置 Microsoft / Google 免费实验渠道，并支持 Google Cloud Translation、Azure Translator、DeepL、DeepLX、百度翻译等自定义渠道。
- **MCP 功能设置**：支持用户自行添加 MCP 名称、类型、URL 与鉴权配置，可在设置中心测试 `listTools()`，并按助手分别启用；OpenAI-compatible 请求会按启用项注入 `tools` 并执行单轮工具调用。
- **连通性测试**：AI 渠道和机器翻译渠道均提供测试入口。
- **设置中心**：完整配置集中在助手窗口中，旧 options 页面只保留兼容壳层。
- **导入导出**：支持导出和导入主要配置、渠道、术语库和样式设置。

> 安全说明：当前 AI 渠道 API Key 存储在 `chrome.storage.sync`，机器翻译渠道密钥也会进入扩展存储。不要在不可信设备或共享浏览器配置中保存高权限密钥。
