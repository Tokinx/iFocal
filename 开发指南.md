# 开发指南

本文档面向 iFocal 的本地开发、调试和维护，包含运行方式、项目结构、架构边界与开发注意事项。

## 快速开始

### 环境要求

- Node.js `^20.19.0` 或 `>=22.12.0`（Vite 8 / `@vitejs/plugin-vue` 要求）
- npm（仓库当前提供 `package-lock.json`）
- Chrome / Edge，开启开发者模式

### 安装与构建

```bash
npm install
npm run build
```

构建产物会输出到 `dist/`，但扩展清单 `manifest.json` 位于项目根目录。因此加载扩展时应选择**项目根目录**，不是 `dist/` 目录。

### 加载到浏览器

1. 打开 `chrome://extensions` 或 `edge://extensions`。
2. 开启“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本仓库根目录，也就是包含 `manifest.json` 的目录。
5. 修改 background / content 相关代码后，需要重新运行构建并在扩展页手动重新加载扩展。

### 首次配置

1. 点击扩展图标或按 `Ctrl+Shift+O` 打开全局助手窗口。
2. 进入设置中心。
3. 在“渠道管理”中添加 AI 渠道，至少填写名称、类型、API Key 和模型列表。
4. 在“机器翻译”中选择默认机器翻译渠道；内置免费渠道可免密使用，但稳定性不适合作为生产承诺。
5. 回到聊天、翻译或总结助手开始使用。

## 常用命令

```bash
npm run dev              # Vite 开发服务器，主要用于窗口页开发
npm run build            # 生产构建
npm run build:watch      # 监听构建，便于扩展调试
npm run preview          # 预览构建页面
npm run check:encoding   # 检查编码
npm run fix:encoding     # 修复编码
npm run ui:add <name>    # 添加 shadcn-vue 组件
npm run ui:add:base      # 添加基础 UI 组件集
npm run ui:add:more      # 添加扩展 UI 组件集
```

当前仓库没有 lint 脚本，也没有测试脚本或单测框架配置。提交改动前至少执行：

```bash
npm run build
npm run check:encoding
```

## 项目结构

```text
iFocal/
├── manifest.json              # MV3 扩展清单，加载扩展时使用根目录
├── window.html                # 全局助手窗口入口，由 Vite 构建到 dist/window.html
├── vite.config.ts             # Vite 多入口构建：window / background / content
├── package.json               # 依赖与 npm 脚本
├── src/
│   ├── background/
│   │   └── index.ts           # Service Worker：窗口管理、模型调用、机器翻译、菜单和快捷键
│   ├── content/
│   │   └── index.ts           # 内容脚本：划词、悬浮、全文翻译和页面 DOM 注入
│   ├── window/
│   │   ├── App.vue            # 全局助手窗口主应用
│   │   ├── router.ts          # hash 路由：聊天 / 翻译 / 总结 / 设置
│   │   ├── pages/             # 助手页面与设置页
│   │   └── components/        # 助手窗口组件与设置中心
│   ├── options/               # 旧设置页兼容壳层，部分 composables 仍被设置中心复用
│   ├── sidebar/
│   │   ├── plugins/ui.ts      # shadcn-vue 组件注册
│   │   └── styles.css         # 共享主题、Tailwind 与组件样式
│   ├── shared/                # 配置、AI prompt、机器翻译、助手、术语库等共享逻辑
│   └── components/ui/         # shadcn-vue 通用组件库
└── scripts/
    └── normalize-encoding.mjs # 编码检查 / 修复脚本
```

## 架构要点

- **三入口构建**：`window.html`、`src/background/index.ts`、`src/content/index.ts` 由 Vite 同时构建，manifest 直接引用 `dist/background.js` 和 `dist/content.js`。
- **AI 请求收敛到 background**：内容脚本和助手窗口不直接请求模型，统一通过 background 选择渠道、模型并处理流式 / 非流式响应。
- **两条 AI 链路**：非流式使用 `chrome.runtime.sendMessage({ action: 'performAiAction' })`；流式使用 `chrome.runtime.connect({ name: 'streaming' })`。
- **配置分层存储**：用户配置、渠道、Prompt、术语库等主要放在 `chrome.storage.sync`；窗口 ID、会话历史、助手配置和运行态选择放在 `chrome.storage.local`。
- **内容脚本自包含**：`src/content/index.ts` 顶部禁止使用顶层 `import`，这是为了避免部分网页环境中的 ESM 注入兼容问题。
- **全文翻译偏机器翻译**：全文翻译按可视区域和队列分批处理，优先走支持 HTML 的机器翻译渠道，不适合作为 AI 高质量全文润色替代品。

## 浏览器权限

`manifest.json` 当前声明：

- `storage`：保存配置、渠道、助手、会话等数据。
- `clipboardRead`：剪贴板监听 / 自动填入。
- `contextMenus`：网页全文翻译右键菜单。
- `tabs`：定位当前标签页、向内容脚本发送消息。
- `<all_urls>`：在网页中注入内容脚本并允许调用翻译接口。

这些权限与产品能力直接相关，但也意味着扩展拥有较宽的网页访问面。开发和分发前应明确告知用户数据处理边界。

## 开发注意事项

- 修改 `src/background/index.ts` 后，需要重新构建并在扩展管理页重新加载扩展。
- 修改 `src/content/index.ts` 后，需要重新构建、重新加载扩展，并刷新目标网页。
- 修改 `src/window/` 的 Vue 页面时，可优先用 `npm run dev` 调试界面，但最终仍要通过扩展环境验证消息链路。
- 渠道模型配置支持 `modelId#displayName`，不要随意改存储格式，否则设置中心和助手窗口会无法正确回显。
- Prompt 模板支持 `{{targetLang}}`、`{{prevLang}}`、`{{text}}` 占位符。
- MCP SDK 已在依赖中，但当前没有面向用户的 MCP 功能入口；不要把它写入已完成特性。
